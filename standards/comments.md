# 注释规范 (Comment Standards)

本文档定义金蝶苍穹开发的代码注释标准。

## 1. XML 文档注释

### 类注释

所有公共类必须有 XML 文档注释。

#### ✅ 正确示例

```csharp
/// <summary>
/// 工作流服务帮助类，提供工作流操作的辅助方法
/// </summary>
/// <remarks>
/// 此类包含启动工作流、提交审批、查询状态等常用方法
/// </remarks>
public class WorkflowServiceHelper
{
    // 类实现
}
```

### 方法注释

所有公共方法必须有 XML 文档注释，包括：
- `<summary>` - 方法说明
- `<param>` - 参数说明
- `<returns>` - 返回值说明
- `<exception>` - 可能抛出的异常

#### ✅ 正确示例

```csharp
/// <summary>
/// 提交单据到工作流进行审批
/// </summary>
/// <param name="billNo">单据编号</param>
/// <param name="billId">单据ID</param>
/// <returns>如果提交成功返回 true，否则返回 false</returns>
/// <exception cref="ArgumentNullException">当 billNo 为 null 时抛出</exception>
/// <exception cref="InvalidOperationException">当单据状态不允许提交时抛出</exception>
public bool SubmitToWorkflow(string billNo, long billId)
{
    // 方法实现
}
```

### 属性注释

公共属性应有简短注释。

```csharp
/// <summary>
/// 单据编号
/// </summary>
public string BillNo { get; set; }
```

## 2. 行内注释

### 何时使用

- 解释复杂的业务逻辑
- 说明非显而易见的实现
- 标记待办事项 (TODO, FIXME)

### 示例

```csharp
public void ProcessBill()
{
    // 根据单据类型决定处理流程
    if (billType == "Invoice")
    {
        // 发票需要校验税号
        ValidateTaxId();
    }

    // TODO: 添加多币种支持
    CalculateAmount();

    // FIXME: 性能优化 - 减少数据库查询次数
    LoadBillDetails();
}
```

### 注释规则

- 使用完整的句子
- 首字母大写
- 中文或英文注释，保持一致
- 避免冗余注释

## 3. 注释标签

### 常用标签

| 标签 | 用途 |
|------|------|
| `<summary>` | 简短描述 |
| `<param>` | 参数说明 |
| `<returns>` | 返回值说明 |
| `<exception>` | 异常说明 |
| `<remarks>` | 详细说明 |
| `<example>` | 使用示例 |
| `<see>` | 参见其他类型 |
| `<seealso>` | 参见其他内容 |

### 示例

```csharp
/// <summary>
/// 数据验证器基类
/// </summary>
/// <remarks>
/// <para>所有验证器应继承此类并实现 Validate 方法。</para>
/// <para>示例：<see cref="FormValidator"/></para>
/// </remarks>
/// <example>
/// <code>
/// var validator = new FormValidator();
/// bool isValid = validator.Validate(data);
/// </code>
/// </example>
public abstract class ValidatorBase
{
    /// <summary>
    /// 验证数据
    /// </summary>
    public abstract bool Validate(object data);
}
```

## 4. 代码块注释

对于复杂的代码块，使用注释说明：

```csharp
public void ProcessWorkflow()
{
    // ==================== 验证阶段 ====================
    ValidateFormData();
    CheckPermissions();

    // ==================== 处理阶段 ====================
    SaveFormData();
    StartWorkflow();

    // ==================== 通知阶段 ====================
    SendNotification();
    LogOperation();
}
```

## 5. 避免的注释

### ❌ 不要注释显而易见的内容

```csharp
// 错误示例
// 获取单据编号
string billNo = GetBillNo();

// 设置值为 10
int count = 10;
```

### ❌ 不要注释掉的代码

```csharp
// 错误示例
// public void OldMethod()
// {
//     // 旧代码
// }
```

正确做法：直接删除，使用版本控制系统追踪历史。

## 6. TODO 和 FIXME

### 格式

```csharp
// TODO: [描述] - [负责人] - [日期]
// FIXME: [问题描述]
```

### 示例

```csharp
// TODO: 添加多语言支持 - 张三 - 2026-05-01
public string GetMessage()
{
    return "操作成功";
}

// FIXME: 高并发下可能出现数据不一致
public void UpdateInventory()
{
    // 实现
}
```
