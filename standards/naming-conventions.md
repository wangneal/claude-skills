# 命名规范 (Naming Conventions)

本文档定义金蝶苍穹开发中类、方法、变量、常量的命名规范。

## 1. 类命名规范

### 规则

- 使用 **PascalCase**（帕斯卡命名法）
- 类名应清晰表达用途和职责
- 避免使用缩写，除非是广泛认可的缩写
- 使用名词或名词短语

### 示例

#### ✅ 正确示例

```csharp
// 使用 PascalCase，清晰表达用途
public class WorkflowServiceHelper
{
    // 类实现
}

public class DataValidator
{
    // 类实现
}

public class DynamicObjectProcessor
{
    // 类实现
}
```

#### ❌ 错误示例

```csharp
// 使用 camelCase
public class workflowServiceHelper { }

// 使用下划线
public class Data_Validator { }

// 使用缩写不清晰
public class WFSHelper { }
```

### 特殊情况

- **异常类**: 以 `Exception` 结尾
  ```csharp
  public class ValidationException : Exception
  ```

- **接口**: 以 `I` 开头
  ```csharp
  public interface IDataService
  ```

- **抽象类**: 可以 `Base` 结尾
  ```csharp
  public abstract class ServiceBase
  ```

## 2. 方法命名规范

### 规则

- 使用 **PascalCase**
- 方法名应清晰表达行为或操作
- 使用动词或动词短语
- 避免使用泛化的名称（如 Do, Process）

### 示例

#### ✅ 正确示例

```csharp
// 使用动词开头，清晰表达操作
public void ValidateFormData()
{
    // 方法实现
}

public DynamicObject LoadBillData(string billNo)
{
    // 方法实现
}

public bool SubmitToWorkflow()
{
    // 方法实现
}
```

#### ❌ 错误示例

```csharp
// 使用 camelCase
public void validateFormData() { }

// 名称不清晰
public void Process() { }

// 过于简短
public void Do() { }
```

### 常见方法命名模式

- **获取数据**: `Get{Entity}`, `Load{Entity}`, `Query{Entity}`
  ```csharp
  public DynamicObject GetFormData()
  public List<DynamicObject> LoadBillList()
  ```

- **保存数据**: `Save{Entity}`, `Update{Entity}`, `Create{Entity}`
  ```csharp
  public void SaveFormData()
  public void UpdateBillStatus()
  ```

- **验证**: `Validate{Entity}`, `Check{Condition}`
  ```csharp
  public bool ValidateFormData()
  public bool CheckPermission()
  ```

- **操作**: `Submit{Entity}`, `Approve{Entity}`, `Delete{Entity}`
  ```csharp
  public void SubmitToWorkflow()
  public void ApproveBill()
  ```

## 3. 变量命名规范

### 规则

- 使用 **camelCase**（驼峰命名法）
- 变量名应具有描述性
- 避免使用单字母变量（循环变量除外）
- 避免使用类型前缀（匈牙利命名法）

### 示例

#### ✅ 正确示例

```csharp
// 局部变量
string billNo = "B001";
DynamicObject formData = new DynamicObject();
List<DynamicObject> billList = new List<DynamicObject>();

// 循环变量
for (int i = 0; i < billList.Count; i++)
{
    DynamicObject bill = billList[i];
}

// 参数
public void ProcessBill(string billNo, DynamicObject billData)
{
    // 方法实现
}
```

#### ❌ 错误示例

```csharp
// 使用 PascalCase
string BillNo = "B001";

// 使用下划线
string bill_no = "B001";

// 使用类型前缀
string strBillNo = "B001";
DynamicObject objFormData = new DynamicObject();

// 使用缩写
DynamicObject bd = new DynamicObject();
```

## 4. 常量命名规范

### 规则

- 使用 **全大写字母**
- 使用下划线分隔单词
- 常量名应完整描述其含义

### 示例

#### ✅ 正确示例

```csharp
// 类常量
public const string DEFAULT_BILL_TYPE = "Invoice";
public const int MAX_RETRY_COUNT = 3;
public const string WORKFLOW_STATUS_APPROVED = "Approved";

// 静态只读字段
public static readonly string CONNECTION_STRING = "...";
```

#### ❌ 错误示例

```csharp
// 使用 camelCase
public const string defaultBillType = "Invoice";

// 使用 PascalCase
public const string DefaultBillType = "Invoice";

// 使用缩写
public const string DFLT_BILL_TYPE = "Invoice";
```

## 5. 字段命名规范

### 规则

- 私有字段使用 camelCase，可加下划线前缀
- 公共属性使用 PascalCase
- 避免使用泛化的名称

### 示例

#### ✅ 正确示例

```csharp
public class BillService
{
    // 私有字段
    private string _billNo;
    private DynamicObject _billData;

    // 公共属性
    public string BillNo { get; set; }
    public DynamicObject BillData { get; set; }

    // 受保护字段
    protected int _status;
}
```

#### ❌ 错误示例

```csharp
public class BillService
{
    // 私有字段使用 PascalCase
    private string BillNo;

    // 公共属性使用 camelCase
    public string billNo { get; set; }

    // 使用类型前缀
    private string strBillNo;
}
```

## 6. 事件命名规范

### 规则

- 使用 PascalCase
- 使用动词或动词短语
- 通常使用现在时或过去时

### 示例

#### ✅ 正确示例

```csharp
// 事件
public event EventHandler FormValidating;     // 正在进行
public event EventHandler FormValidated;      // 已完成

public event EventHandler<BillEventArgs> BillSaving;
public event EventHandler<BillEventArgs> BillSaved;
```

## 注意事项

1. **一致性**: 整个项目中保持命名风格一致
2. **可读性**: 名称应自解释，避免需要额外注释才能理解
3. **上下文**: 考虑使用场景，避免过度冗长
4. **团队约定**: 如有特殊约定，优先遵循团队规范
