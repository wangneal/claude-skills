- 🚀 **5阶段工作流** - 新建→讨论→计划→执行→验证
- 🔍 **统一搜索** - 整合SDK文档与社区知识库 (18,077条)

## 🚀 工作流

### 5阶段开发流程

```
kd-new-project     → Phase 1-2: 新建项目 + 理解项目
kd-discuss <N>     → Phase 3:   需求讨论 → CONTEXT.md
kd-plan <N>        → Phase 4:   生成计划 → PLAN.md
kd-execute <N>     → Phase 5:   执行开发
kd-verify <N>      → Phase 6:   验证UAT
```

### 快速开始

```bash
# 1. 新建项目（自动理解项目结构）
kingdee-dev-plugin kd-new-project

# 2. 需求讨论（分析+讨论需求）
kingdee-dev-plugin kd-discuss 1

# 3. 生成执行计划
kingdee-dev-plugin kd-plan 1

# 4. 执行开发
kingdee-dev-plugin kd-execute 1

# 5. 验证UAT
kingdee-dev-plugin kd-verify 1

# 自动推进到下一阶段
kingdee-dev-plugin kd-next
```

## 📖 命令参考

### 工作流命令

| 命令 | 说明 |
|------|------|
| `kd-new-project` | 新建项目+理解项目结构 |
| `kd-discuss <N>` | 需求讨论：分析+讨论需求 |
| `kd-plan <N>` | 生成计划：规格+计划 |
| `kd-execute <N>` | 执行开发：执行计划任务 |
| `kd-verify <N>` | 验证UAT：需求校验+验收测试 |
| `kd-next` | 自动推进到下一阶段 |

### 辅助命令

| 命令 | 说明 |
|------|------|
| `kd-analyze <需求>` | 需求分析：搜索+推荐+SDK+约束 |
| `kd-search <关键词>` | 统一搜索SDK和社区内容 |
| `kd-recommend <需求>` | 智能推荐SDK API和最佳实践 |
| `kd-gen <类名>` | 生成SDK约束文件 |
| `kd-list [模块]` | 列出SDK模块和类 |
