# bos-flow 模板使用指南

本目录包含四阶段工作流的所有模板文件。

## 模板列表

| 模板文件 | 阶段 | 用途 |
|---------|------|------|
| PLANNING.md | 计划阶段 | 功能分析、SDK 调研、任务拆分 |
| DEVELOPMENT.md | 开发阶段 | 代码实现、单元测试、集成测试 |
| TESTING.md | 测试阶段 | 功能测试、Bug 修复、回归测试 |
| UAT.md | UAT 验收 | 验收标准、测试用例、交付文档 |

## 如何使用模板

### 1. 计划阶段模板 (PLANNING.md)

**使用时机:** 项目启动时，在开发前进行规划。

**填写步骤:**

1. **功能分析**
   - 从需求文档中提取业务需求
   - 定义功能范围（包含/不包含）
   - 编写用户场景

2. **SDK 调研**
   - 使用 `bos-flow search <关键词>` 搜索相关 SDK
   - 记录相关类和方法
   - 编写示例代码

3. **任务拆分**
   - 将功能分解为具体任务
   - 定义任务依赖关系
   - 评估风险

**示例命令:**

```bash
# 搜索 DynamicObject 相关文档
bos-flow search DynamicObject

# 搜索工作流相关方法
bos-flow search WorkflowServiceHelper
```

**注意事项:**

- 确保所有功能都有对应的任务
- SDK 调研要充分，避免后期发现技术问题
- 风险评估要诚实，不要隐瞒风险

### 2. 开发阶段模板 (DEVELOPMENT.md)

**使用时机:** 计划阶段完成后，开始编码实现时使用。

**填写步骤:**

1. **开发计划**
   - 将计划阶段的任务分配给开发人员
   - 设定任务开始和结束日期
   - 定义开发里程碑

2. **遵循代码规范**
   - 按照命名规范编写代码
   - 添加必要的注释
   - 正确处理异常
   - 参考 SDK 使用规范

3. **编写单元测试**
   - 为所有业务逻辑编写单元测试
   - 确保测试覆盖率 >= 80%
   - 测试边界条件和异常分支

4. **执行集成测试**
   - 在测试环境中执行集成测试
   - 记录测试结果
   - 修复发现的问题

5. **代码审查**
   - 完成代码后进行代码审查
   - 使用代码审查清单
   - 记录审查发现的问题

**注意事项:**

- 严格遵循代码规范，保持代码一致性
- 单元测试必须和代码同步完成
- 集成测试要在真实环境中执行
- 代码审查不可省略

**SDK 使用提示:**

```bash
# 查找相关 SDK 类
bos-flow search [类名]

# 查找特定方法
bos-flow search [方法名]
```

### 3. 测试阶段模板 (TESTING.md)

**使用时机:** 开发阶段完成后，进行全面测试时使用。

**填写步骤:**

1. **制定测试计划**
   - 明确测试范围
   - 配置测试环境
   - 准备测试数据
   - 设定测试进度

2. **编写测试用例**
   - 为每个功能编写测试用例
   - 定义测试步骤和预期结果
   - 准备测试数据
   - 执行测试并记录结果

3. **Bug 跟踪**
   - 记录所有发现的 Bug
   - 指派给开发人员修复
   - 验证修复结果
   - 统计 Bug 情况

4. **执行回归测试**
   - Bug 修复后执行回归测试
   - 确保修复没有引入新问题
   - 记录回归测试结果

5. **编写测试总结**
   - 汇总测试完成情况
   - 评估遗留风险
   - 给出测试结论

**注意事项:**

- 测试用例要覆盖所有功能点
- Bug 记录要详细，便于复现
- 回归测试不可省略
- 遗留 Bug 要明确风险

**测试数据准备:**

使用 SDK 创建测试数据：

```bash
# 查找数据创建相关 SDK
bos-flow search Create
bos-flow search Save
```

### 4. UAT 验收模板 (UAT.md)

**使用时机:** 测试阶段完成后，用户验收时使用。

**填写步骤:**

1. **准备验收材料**
   - 整理功能清单
   - 准备验收标准
   - 准备 UAT 测试用例

2. **定义验收标准**
   - 功能验收标准：每个功能的具体验收条件
   - 质量验收标准：代码、测试、文档的质量要求
   - 性能验收标准：响应时间、并发等性能指标

3. **执行 UAT 测试**
   - 按照用户场景执行测试
   - 记录测试结果
   - 记录发现的问题

4. **准备交付文档**
   - 用户手册
   - 技术文档
   - 部署文档
   - 测试报告
   - API 文档

5. **完成验收确认**
   - 填写验收结论
   - 双方签字确认
   - 处理遗留问题

**注意事项:**

- 验收标准要明确、可测量
- UAT 测试要覆盖真实业务场景
- 交付文档要完整、准确
- 遗留问题要明确责任人和完成时间

**验收成功标志:**

- 所有验收标准满足
- UAT 测试通过
- 交付文档完整
- 双方签字确认

## 模板位置

- 模板目录: `.bos-flow/templates/`
- SDK 知识库: `.bos-flow/sdk/`
- 项目配置: `.bos-flow/config.json`

## 下一步

完成计划阶段模板后，运行：

```bash
bos-flow next
```

进入开发阶段。
